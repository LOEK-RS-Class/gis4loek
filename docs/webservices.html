Webdienste für Karten und Geodaten

Über Webdienste können Karten und Daten direkt bezogen werden, ohne diese wirklich herunterzuladen. Stell dir einen Musikstreaming Anbieter vor - auch hier werden die Songs nicht dauerhaft auf dem Endgerät gespeichert, sondern bei Bedarf angefragt und heruntergeladen. In so einem Webdienst gibt es zwei Akteure: ein Provider / Server der die Karten und Daten bereitstellt, und ein Client der die Karten und Daten anfragt. Damit die Kommunikation zwischen Server und Client funktionert, dass also die gewünschten Daten auch beim Client ankommen müssen sich beide auf eine gemeinsame Sprache einigen. Diese Kommunikation wird in sogenannten Protokollen definiert und als Standards des OGC (Open Geospatial Consortium) veröffentlicht. Dabei gibt es mehrere zur Auswahl:

Web Map Service - WMS

Beim Web Map Service (ogc.org/standards/wms/) liefert der Server Bildausschnitte, in der Regel PNG, für die vom Clienten angefragte Koordinaten und Auflösungen. Folgende Anfragen sind das Minimum was ein WMS Standard liefern muss:

  • getCapabilities zur Anfrage von Metadaten
  • getMap zur Anfrage von Karten

Ein Beispiel für getCapabilities:

https://sgx.geodatenzentrum.de/wms_basemapde_schummerung?request=GetCapabilities&service=WMS

Die getMap URLs sind etwas Umfangreicher und setzen sich zusammen aus Serveradresse, Koordinaten, Bildgröße und Format:

https://sgx.geodatenzentrum.de/wms_basemapde_schummerung?service=wms&version=1.3.0&request=GetMap&Layers=de_basemapde_web_raster_combshade&STYLES=default&CRS=EPSG:25832&bbox=500000,5700000,500200,5700200&width=150&Height=150&Format=image/png

Web Map Tile Service - WMTS

Der Web Map Tile Service ist eine Variante des WMS bei dem die Daten vorprozessiert gekachelt (tiles) auf dem Server geordnet sind. Dadurch wird die Anfrage performanter, benötigt aber auch mehr Speicherplatz. https://www.ogc.org/standards/wmts/

https://sgx.geodatenzentrum.de/wmts_basemapde_schummerung/tile/1.0.0/de_basemapde_web_raster_combshade/default/DE_EPSG_3857_ADV/06/662/1082.png

Web Feature Service - WFS

Im Gegensatz zu WM(T)S werden beim Web Feature Service tatsächliche Geodaten im Vektorformat übertragen. Auch hier werden die Daten nur nach und nach, je nach Kartenausschnitt heruntergeladen. Dabei werden drei Anfragen zwingend vorgegeben:

  • getCapabilities zur Anfrage von allgemeinen Metadaten
  • DescribeFeatureType zur Anfrage von Metadaten spezifischer Vektoren
  • getFeature zur Anfrage der Vektor Objekte, also die tatsächlichen Daten

Beispielhaft eine getFeature Anfrage eines Features (also ein Vector Objekt mit entprechenden Attributen)

https://www.stadt-muenster.de/ows/mapserv706/odgruenserv?REQUEST=GetFeature&SERVICE=WFS&VERSION=2.0.0&TYPENAME=ms:Gruenflaechen&EXCEPTIONS=XML&MAXFEATURES=1
